# 스크린샷 촬영용 샘플 데이터

## 📋 회원가입 폼 입력값

스크린샷 촬영을 위해 사용할 샘플 데이터입니다.

### 자동 입력 기능 (개발 모드 전용)
- **디버그 모드에서만** 회원가입/로그인 화면 오른쪽 상단에 **✨ 아이콘 버튼** 표시
- 릴리즈 빌드에서는 버튼이 보이지 않습니다 (스크린샷 촬영 시 버튼 없음)
- 버튼을 클릭하면 자동으로 폼이 채워집니다

**참고**: 스크린샷 촬영은 릴리즈 모드(`flutter run --release`)에서 하면 버튼이 보이지 않습니다.

### 수동 입력 값

#### 회원가입 화면
- **이름**: `테스트 사용자`
- **이메일**: `test@example.com`
- **비밀번호**: `password123`

#### 로그인 화면
- **이메일**: `test@example.com`
- **비밀번호**: `password123`

## 📸 스크린샷 촬영 순서

### 1단계: 회원가입 화면
1. 앱 실행
2. "계정이 없으신가요? 회원가입" 클릭
3. ✨ 아이콘 버튼 클릭 (자동 입력) 또는 수동 입력
4. 폼이 채워진 상태에서 스크린샷 촬영

### 2단계: 로그인 화면
1. "이미 계정이 있으신가요? 로그인" 클릭
2. ✨ 아이콘 버튼 클릭 (자동 입력) 또는 수동 입력
3. 폼이 채워진 상태에서 스크린샷 촬영

### 3단계: 게시글 목록 화면
1. 로그인 후
2. 샘플 데이터 생성 (메뉴 > 샘플 데이터 생성)
3. 게시글 목록 스크린샷 촬영

### 4단계: 게시글 상세 화면
1. 게시글 클릭
2. 상세 내용 확인
3. 스크린샷 촬영

### 5단계: 댓글 화면
1. 게시글 상세 화면에서
2. 댓글 섹션 확인
3. 스크린샷 촬영

## 💡 팁

1. **폼 채우기 옵션**:
   - ✅ **채워진 폼 (권장)**: 실제 사용 흐름을 보여주어 더 전문적으로 보입니다
   - ✅ **빈 폼**: 간단하고 깔끔한 느낌, 사용자가 입력할 준비가 된 상태를 보여줍니다
   - 두 가지 모두 가능하므로 취향에 따라 선택하세요

2. **실제 회원가입**: 실제로 회원가입을 진행하지 않아도 스크린샷 촬영이 가능합니다.

3. **데이터 일관성**: 모든 스크린샷에서 동일한 샘플 데이터를 사용하면 일관된 느낌을 줄 수 있습니다.

## 📸 스크린샷 스타일 추천

### 스타일 1: 채워진 폼 (권장)
- 실제 사용 흐름을 보여줌
- 더 전문적인 느낌
- 사용자가 어떻게 입력하는지 시각적으로 이해하기 쉬움

**예시:**
- 이름: `테스트 사용자`
- 이메일: `test@example.com`
- 비밀번호: `•••••••••` (마스킹 처리됨)

### 스타일 2: 빈 폼
- 간단하고 깔끔한 느낌
- 사용자가 입력할 준비가 된 상태
- 힌트 텍스트가 잘 보임

**예시:**
- 이름: (빈 필드)
- 이메일: `example@email.com` (힌트만 표시)
- 비밀번호: (빈 필드)

## 📝 추가 정보

### 샘플 데이터 특징
- 이메일: `test@example.com` (실제 이메일이 아니어도 입력 가능)
- 비밀번호: `password123` (최소 6자 이상)
- 이름: `테스트 사용자` (한글 이름)

### 주의사항
- 실제 회원가입을 진행하려면 유효한 이메일 주소가 필요합니다.
- 스크린샷 촬영만을 위한 경우 자동 입력 기능을 사용하세요.
