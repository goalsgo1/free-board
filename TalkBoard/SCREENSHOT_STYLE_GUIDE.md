# 스크린샷 스타일 가이드

## 📸 입력 폼 스크린샷 스타일

### 선택 옵션

스크린샷 촬영 시 입력 폼을 채울지 말지는 **선택 사항**입니다. 두 가지 스타일 모두 유효합니다.

## 스타일 1: 채워진 폼 (권장) ✅

### 장점
- ✅ 실제 사용 흐름을 보여줌
- ✅ 더 전문적인 느낌
- ✅ 사용자가 어떻게 입력하는지 시각적으로 이해하기 쉬움
- ✅ 완료된 상태를 보여줌

### 단점
- ❌ 개인정보처럼 보일 수 있음 (하지만 샘플 데이터이므로 문제 없음)

### 촬영 방법
1. 릴리즈 모드로 앱 실행 (`flutter run --release`)
2. 회원가입 화면으로 이동
3. 수동으로 입력:
   - 이름: `테스트 사용자`
   - 이메일: `test@example.com`
   - 비밀번호: `password123` (입력 시 자동으로 마스킹 처리됨)
4. 스크린샷 촬영

## 스타일 2: 빈 폼 (대안) ✅

### 장점
- ✅ 간단하고 깔끔한 느낌
- ✅ 사용자가 입력할 준비가 된 상태를 보여줌
- ✅ 힌트 텍스트가 잘 보임
- ✅ 개인정보 걱정 없음

### 단점
- ❌ 실제 사용 흐름을 보여주지 않음

### 촬영 방법
1. 릴리즈 모드로 앱 실행 (`flutter run --release`)
2. 회원가입 화면으로 이동
3. **아무것도 입력하지 않음** (빈 상태 유지)
4. 스크린샷 촬영

## 🎯 추천

### 일반적인 추천: **채워진 폼**
대부분의 앱 스토어에서는 채워진 폼 스크린샷을 사용합니다. 실제 사용 흐름을 보여주어 더 전문적으로 보입니다.

### 대안: **빈 폼**
특히 개인정보 보호에 민감한 경우나, 더 미니멀한 느낌을 원할 때 빈 폼을 사용할 수 있습니다.

## 📋 앱 스토어별 관례

### Google Play Store
- **일반적**: 채워진 폼 (실제 사용 흐름)
- **허용**: 빈 폼도 가능

### Apple App Store
- **일반적**: 채워진 폼 (실제 사용 흐름)
- **허용**: 빈 폼도 가능

## 💡 팁

1. **일관성 유지**: 모든 스크린샷에서 동일한 스타일 사용 권장
2. **비밀번호 필드**: 입력하면 자동으로 마스킹 처리되므로 안전합니다
3. **이름 필드**: 한글 이름을 사용하면 한국 사용자에게 친숙합니다

## ✅ 체크리스트

### 채워진 폼으로 촬영하는 경우
- [ ] 릴리즈 모드로 실행 (`flutter run --release`)
- [ ] 이름 입력: `테스트 사용자`
- [ ] 이메일 입력: `test@example.com`
- [ ] 비밀번호 입력: `password123` (마스킹됨)
- [ ] 스크린샷 촬영

### 빈 폼으로 촬영하는 경우
- [ ] 릴리즈 모드로 실행 (`flutter run --release`)
- [ ] 입력하지 않음 (빈 상태 유지)
- [ ] 힌트 텍스트가 잘 보이는지 확인
- [ ] 스크린샷 촬영

## 🎨 최종 권장사항

**채워진 폼을 권장합니다**, 하지만 두 가지 모두 가능하므로:
- 선호하는 스타일 선택
- 모든 스크린샷에서 일관성 유지
- 실제 앱과 가장 비슷한 느낌이 나도록 선택
