# 📸 갤럭시 탭 7 Lite 화면 캡처 방법

## 🎯 방법 1: 물리 버튼 사용 (가장 일반적) ⭐

### 버튼 조합
**전원 버튼 + 볼륨 다운 버튼**을 **동시에** 약 1초간 누르기

### 위치 확인
- **전원 버튼**: 갤럭시 탭 7 Lite의 오른쪽 가장자리 또는 상단
- **볼륨 다운 버튼**: 전원 버튼 근처 (오른쪽 가장자리 또는 상단)

### 촬영 방법
1. 원하는 화면으로 이동
2. **전원 버튼**과 **볼륨 다운 버튼**을 **동시에** 누름
3. 약 1초간 유지
4. 화면이 깜빡이거나 사운드가 나면 성공

### 저장 위치
- 스크린샷은 자동으로 **갤러리 → Screenshots** 폴더에 저장됩니다

---

## 🎯 방법 2: 손바닥 제스처 (삼성 고유 기능)

### 활성화 방법
1. **설정** → **고급 기능**
2. **모션 및 제스처** → **손바닥으로 화면 캡처**
3. **켜기** 활성화

### 사용 방법
1. 원하는 화면으로 이동
2. 손바닥의 **측면**을 화면 **왼쪽에서 오른쪽으로** (또는 오른쪽에서 왼쪽으로) 스와이프
3. 화면이 깜빡이면 성공

---

## 🎯 방법 3: S Pen 사용 (S Pen 모델인 경우)

### 사용 방법
1. S Pen을 꺼냄
2. **Air Command** 메뉴에서 **스마트 선택** 또는 **화면 캡처** 선택
3. 원하는 영역 선택

---

## 🎯 방법 4: 빠른 설정 패널 사용

### 사용 방법
1. 화면 **상단에서 아래로** 스와이프하여 알림 패널 열기
2. **빠른 설정** 아이콘 클릭 (또는 두 번 아래로 스와이프)
3. **화면 캡처** 아이콘 찾아서 클릭

## 💻 ADB로 PC로 바로 복사하기 (개발용)

### 사전 준비
1. **개발자 옵션** 활성화:
   - 설정 → 휴대전화 정보 → 소프트웨어 정보
   - **빌드 번호**를 7번 빠르게 탭
2. **USB 디버깅** 활성화:
   - 설정 → 개발자 옵션
   - **USB 디버깅** 켜기
3. USB로 태블릿을 Mac에 연결

### ADB 명령어로 촬영 및 복사

```bash
cd /Users/haemin/Documents/GitHub/project/TalkBoard

# 1. 디바이스 연결 확인
adb devices

# 2. 스크린샷 촬영 (태블릿 내부 저장)
adb shell screencap -p /sdcard/screenshot.png

# 3. PC로 복사
adb pull /sdcard/screenshot.png screenshots/android/tablet_01_post_list.png

# 4. 태블릿에서 임시 파일 삭제 (선택)
adb shell rm /sdcard/screenshot.png
```

### 한 줄 명령어
```bash
adb shell screencap -p /sdcard/screenshot.png && adb pull /sdcard/screenshot.png screenshots/android/tablet_01_post_list.png && adb shell rm /sdcard/screenshot.png
```

## 💡 팁

### 물리 버튼 사용 시
- **주의**: 전원 버튼과 볼륨 다운 버튼을 **동시에** 눌러야 합니다
- **시간**: 약 1초간 누르고 있으면 됩니다
- **실패 시**: 너무 오래 누르면 전원 메뉴가 나타날 수 있습니다

### ADB 사용 시
- **USB 연결**: 태블릿을 USB로 Mac에 연결
- **USB 디버깅 허용**: 태블릿 화면에서 "USB 디버깅을 허용하시겠습니까?" → **허용** 클릭
- **파일 경로**: PC로 복사한 후 `screenshots/android/` 폴더에 저장

### 저장 위치 확인
- **태블릿 내부**: 갤러리 앱 → Screenshots 폴더
- **PC로 복사**: ADB pull 명령어로 원하는 위치에 저장

## ⚠️ 문제 해결

### 물리 버튼으로 캡처가 안 될 때
- **방법 1**: 손바닥 제스처 사용
- **방법 2**: 빠른 설정 패널의 화면 캡처 버튼 사용
- **방법 3**: ADB 명령어 사용

### ADB 연결이 안 될 때
```bash
# ADB 서버 재시작
adb kill-server
adb start-server

# 디바이스 재연결
adb devices

# 태블릿에서 USB 디버깅 허용 확인
```
