# 데이터 보안 설정 가이드

## 📋 톡보드 앱 데이터 수집 현황

톡보드 앱은 다음 데이터를 수집합니다:

### 수집하는 데이터
1. **사용자 인증 정보**
   - 이메일 주소 (Firebase Authentication)
   - 비밀번호 (암호화되어 저장, Firebase 관리)

2. **사용자 생성 콘텐츠**
   - 게시글 제목 및 내용
   - 댓글 내용
   - 작성자 이름 (닉네임)

3. **사용자 ID**
   - Firebase UID (사용자 고유 식별자)

4. **앱 활동 데이터**
   - 게시글 조회수
   - 게시글 작성/수정/삭제 시간
   - 댓글 작성/수정/삭제 시간

### 질문 2: "앱에서 수집하는 모든 사용자 데이터를 암호화하여 전송하나요?"

**답변: "예"** 선택 ✅

**이유:**
- Firebase는 모든 데이터를 **HTTPS**로 암호화하여 전송합니다
- Firestore와 Authentication은 기본적으로 암호화 전송을 사용합니다
- 네트워크를 통한 모든 데이터는 암호화됩니다

**필요 작업:**
1. **"예"** 라디오 버튼 선택

---

### 질문 3: "다음 중 앱에서 지원하는 계정 생성 방법은 무엇인가요?"

**답변: "사용자 이름 및 비밀번호"** 선택 ✅

**이유:**
- 톡보드 앱은 **이메일/비밀번호** 방식으로 계정을 생성합니다
- Firebase Authentication의 이메일/비밀번호 인증을 사용합니다
- 사용자가 앱 내에서 직접 계정을 생성할 수 있습니다

**필요 작업:**
1. **"사용자 이름 및 비밀번호"** 체크박스 선택

**선택 안 함:**
- 사용자 이름 및 기타 인증 ❌ (비밀번호 사용)
- 사용자 이름, 비밀번호, 기타 인증 ❌ (기타 인증 없음)
- OAuth ❌ (소셜 로그인 없음)
- 기타 ❌ (해당 없음)
- 앱에서 사용자가 계정을 만들도록 허용하지 않음 ❌ (계정 생성 가능)

## 🔍 다음 단계 안내

"예" 선택 후 다음 단계:

### 1단계: 데이터 유형 선택
톡보드 앱에서 수집하는 데이터 유형을 선택해야 합니다:

**필수 선택 항목:**
- [ ] **개인 식별 정보 (Personal identifiers)**
  - 이메일 주소
  - 사용자 ID (Firebase UID)

- [ ] **사용자 생성 콘텐츠 (User-generated content)**
  - 게시글 (게시글 제목, 내용)
  - 댓글 (댓글 내용)

- [ ] **앱 활동 (App activity)**
  - 앱 상호작용 (게시글 작성/수정/삭제, 댓글 작성/수정/삭제)
  - 조회수

**선택 안 함:**
- 위치 정보 ❌ (수집 안 함)
- 연락처 정보 ❌ (수집 안 함)
- 사진/비디오 ❌ (수집 안 함)
- 금융 정보 ❌ (수집 안 함)
- 건강 정보 ❌ (수집 안 함)

### 2단계: 데이터 취급 방식
- 데이터 저장 위치: Firebase Firestore (Google Cloud)
- 데이터 암호화: Firebase 자동 암호화
- 데이터 공유: 제3자와 공유하지 않음

### 3단계: 미리보기
- 설정한 내용 검토
- 확인 후 제출
