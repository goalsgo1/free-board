# 🪟 모달 및 팝업 화면 분석 및 디자인

## 📋 개요

모든 화면을 분석하여 팝업(모달)이 필요한 부분과 별도 화면이 필요한 부분을 정리하고, 각 팝업의 디자인을 정의합니다.

---

## 🎯 팝업 vs 별도 화면 판단 기준

### 팝업(모달)이 적합한 경우
1. **간단한 확인/취소 작업**: 삭제 확인, 로그아웃 확인 등
2. **짧은 입력 작업**: 금액 입력, 간단한 텍스트 입력 등
3. **선택 작업**: 결제 수단 선택, 카테고리 선택 등
4. **빠른 조회**: 이미지 확대 보기, 간단한 정보 표시
5. **현재 화면 컨텍스트 유지**: 현재 화면을 벗어나지 않고 빠르게 처리

### 별도 화면이 적합한 경우
1. **복잡한 입력 작업**: 여러 단계 입력, 긴 텍스트 입력 등
2. **목록 조회**: 내역 목록, 통계 그래프 등
3. **설정 화면**: 여러 옵션을 설정하는 화면
4. **상세 정보**: 상세한 정보를 보여주는 화면
5. **다단계 프로세스**: 여러 단계를 거치는 작업

### 1. 삭제 확인 팝업

#### 사용 위치
- 추모관 목록: 추모관 삭제
- 추모관 상세: 댓글 삭제
- 감정 공유: 글 삭제, 댓글 삭제
- 삭제된 추모관 복원: 영구 삭제

#### 디자인
```
┌─────────────────────────────────────────┐
│                                         │
│         ⚠️                              │
│                                         │
│      삭제하시겠습니까?                    │
│                                         │
│  이 작업은 되돌릴 수 없습니다.            │
│                                         │
│  ┌─────────────┐  ┌─────────────┐     │
│  │   취소       │  │   삭제       │     │
│  └─────────────┘  └─────────────┘     │
│                                         │
└─────────────────────────────────────────┘
```

#### 반응형 규칙
- **작은 화면**: 전체 너비 - 32px, 최대 높이: 화면 높이의 50%
- **제목**: 16px, 최대 2줄
- **내용**: 14px, 최대 3줄
- **버튼**: 전체 너비, 높이 48px, 세로 배치 (작은 화면)

## 📐 팝업 디자인 공통 규칙

### 레이아웃
- **최대 너비**: 화면 너비 - 32px (작은 화면: 24px)
- **최대 높이**: 화면 높이의 80-90% (내용에 따라 조정)
- **배경**: 반투명 오버레이 (Colors.black54)
- **모서리**: BorderRadius.circular(8-12px)
- **패딩**: 16px (작은 화면: 12px)

### AppBar
- **높이**: 56px
- **닫기 버튼**: 왼쪽 상단 또는 오른쪽 상단
- **제목**: 중앙 정렬, 18px, 최대 1줄

### 입력 필드
- **높이**: 44px (최소)
- **폰트 크기**: 16px (iOS 자동 줌 방지)
- **패딩**: 좌우 12px, 상하 12px
- **여백**: 하단 16px

### 버튼
- **높이**: 48px
- **폰트 크기**: 14-16px
- **여백**: 버튼 간 8px
- **작은 화면**: 전체 너비, 세로 배치

### 스크롤
- **SingleChildScrollView**: 내용이 많을 경우 사용
- **ListView**: 목록이 많을 경우 사용

### 애니메이션
- **표시**: FadeTransition + ScaleTransition
- **닫기**: FadeTransition + ScaleTransition
- **지속 시간**: 200-300ms

## 📝 참고사항

1. **팝업은 현재 화면 컨텍스트를 유지**하면서 빠르게 작업을 완료할 수 있도록 설계
2. **별도 화면은 복잡한 작업**이나 상세한 정보를 보여줄 때 사용
3. **모든 팝업은 반응형 레이아웃** 규칙을 따라야 함
4. **팝업 닫기는 뒤로가기 버튼, 오버레이 탭, 닫기 버튼** 모두 지원
5. **팝업 내부에서도 스크롤**이 필요할 경우 `SingleChildScrollView` 사용
