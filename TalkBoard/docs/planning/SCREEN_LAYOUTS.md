# 📐 화면 레이아웃 디자인 및 화면 이동 플로우

## 📱 반응형 레이아웃 가이드

모든 화면은 다양한 모바일 기기 크기에서도 레이아웃이 깨지지 않도록 반응형으로 설계되어야 합니다.

### 반응형 레이아웃 원칙
- **최소 화면 크기**: 320px × 568px (iPhone SE)
- **텍스트 오버플로우 방지**: 모든 텍스트는 `maxLines`와 `TextOverflow.ellipsis` 사용
- **제목 잘림 방지**: 제목은 최대 2줄까지, 필요시 "..." 표시
- **입력 필드**: 최소 높이 44px, 폰트 크기 최소 16px
- **버튼**: 작은 화면에서는 전체 너비 사용
- **이미지**: 화면 너비를 초과하지 않도록, 비율 유지
- **스크롤**: 모든 화면에서 `SingleChildScrollView` 사용

**자세한 반응형 레이아웃 가이드**: [RESPONSIVE_LAYOUT.md](./RESPONSIVE_LAYOUT.md) 참고

### 진입 경로
- **로그인 후 자동 이동**: 회원가입/로그인 완료 시
- **하단 네비게이션**: 홈 아이콘 클릭

### 사용 가능한 기능
1. **기념일 알림 카드 클릭** → 추모관 상세 화면 이동
2. **추천 추모글 카드 클릭** → 추모관 상세 화면 이동
3. **최근 등록된 추모관 카드 클릭** → 추모관 상세 화면 이동
4. **후회 없는 편지 섹션 클릭** → 후회 없는 편지 화면 이동
5. **애완동물 추모관 만들기 버튼 클릭** → 추모관 생성 화면 (애완동물 타입 선택)
6. **하단 네비게이션**:
   - 📚 추모관 목록 화면 이동
   - 🕊️ 기도 요청 화면 이동
   - 💬 감정 공유 게시판 이동 (탭: 감정 공유 / 후회 없는 편지)
   - 👤 마이페이지 이동

### 반응형 레이아웃 규칙

#### 작은 화면 (320px ~ 375px)
- **AppBar 제목**: 16px, 최대 1줄, `TextOverflow.ellipsis`
- **앱의 목적과 마인드 섹션**:
  - 제목: 16px, 최대 1줄
  - 메인 메시지: 14px, 최대 2줄
  - 서브 메시지: 12px, 최대 2줄
  - 패딩: 12px
  - `Flexible` 위젯 사용
- **기념일 알림 카드**:
  - 높이: 100px (고정)
  - 패딩: 12px
  - 제목: 14px, 최대 1줄
  - 날짜: 12px
  - 버튼: 전체 너비
- **추모글 카드**:
  - 높이: 180px (고정)
  - 이미지: 100px × 100px (고정)
  - 제목: 14px, 최대 1줄, `TextOverflow.ellipsis`
  - 내용: 12px, 최대 2줄, `TextOverflow.ellipsis`
  - `Flexible` 위젯 사용
- **최근 등록된 추모관**:
  - 카드 너비: 120px (고정)
  - 이미지: 80px × 80px (고정)
  - 이름: 12px, 최대 1줄, `TextOverflow.ellipsis`
- **애완동물 추모관 안내**:
  - 섹션 패딩: 24px
  - 제목: 20px, 최대 2줄, `TextOverflow.ellipsis`
  - 메인 메시지: 16px, 최대 2줄
  - 애완동물 아이콘: 4개 가로 배치, 각각 `Expanded` 사용
  - 설명 텍스트: 14px, 최대 2줄, `TextOverflow.ellipsis`
  - 버튼: 전체 너비, 높이 48px
- **앱 기능 안내**:
  - 아이템 높이: 60px
  - 아이콘: 24px
  - 텍스트: 12px, 최대 1줄

#### 중간 화면 (375px ~ 414px)
- **AppBar 제목**: 18px
- **앱의 목적과 마인드 섹션**: 패딩 16px
- **기념일 알림 카드**: 높이 120px, 패딩 16px
- **애완동물 추모관 안내**: 패딩 20px, 제목 22px
- **추모글 카드**: 높이 200px, 이미지 120px × 120px
- **최근 등록된 추모관**: 카드 너비 140px

#### 큰 화면 (414px+)
- **추모글 카드**: 높이 220px, 이미지 140px × 140px

#### 공통 규칙
- **전체 레이아웃**: `SingleChildScrollView` 사용
- **카드**: `mainAxisSize: MainAxisSize.min` 사용
- **텍스트**: `Flexible` 또는 `Expanded` 사용
- **여백**: 좌우 16px (작은 화면: 12px)

4. **댓글 작성** → 댓글 입력창에 텍스트 입력 → 전송
5. **댓글 수정/삭제** → 작성자만 가능
6. **후회 없는 편지 탭** → 이 추모관과 연결된 후회 없는 편지 표시
7. **AI 기억 답장 탭** → 대화 업로드, AI 답장 생성, 저장된 답장 조회
8. **추모금 전달** → 추모금 전달 모달 열기 → 결제 (사람 추모관에만 표시)

### 반응형 레이아웃 규칙

#### 작은 화면 (320px ~ 375px)
- **AppBar 제목**: 16px, 최대 1줄, `TextOverflow.ellipsis`
- **고인 이름**: 18px, 최대 1줄, `TextOverflow.ellipsis`
- **생애 요약**: 12px, 최대 3줄, `TextOverflow.ellipsis`
- **영원한 시간 섹션**:
  - 패딩: 12px
  - 제목: 14px, 최대 1줄
  - 시간: 12px, 최대 2줄
  - 메시지: 12px, 최대 1줄
- **사진 갤러리**: 2열 그리드, 이미지 크기: (화면 너비 - 48px) / 2
- **동영상**: 전체 너비 - 32px, 높이: (너비 × 9 / 16)
- **편지 영역**:
  - 최대 높이: 화면 높이의 40%
  - 스크롤 가능 (`SingleChildScrollView`)
  - 텍스트: 12px, 줄 간격 1.5
- **댓글 카드**:
  - 패딩: 12px
  - 작성자: 12px, 최대 1줄
  - 내용: 12px, 최대 3줄, `TextOverflow.ellipsis`
- **댓글 입력창**: 전체 너비 - 32px, 높이 44px, 폰트 16px
- **하단 버튼**: 전체 너비, 세로 배치 (작은 화면)
- **방문 수**: 12px

#### 중간 화면 (375px ~ 414px)
- **고인 이름**: 20px, 최대 2줄
- **생애 요약**: 14px, 최대 4줄
- **사진 갤러리**: 3열 그리드
- **댓글 카드**: 패딩 16px
- **하단 버튼**: 가로 배치 가능

#### 큰 화면 (414px+)
- **사진 갤러리**: 4열 그리드
- **편지 영역**: 최대 높이 50%

#### 공통 규칙
- **전체 레이아웃**: `SingleChildScrollView` 사용
- **모든 텍스트**: `Flexible` 또는 `Expanded` 사용
- **이미지**: `BoxFit.cover` 또는 `BoxFit.contain` 사용
- **버튼**: 최소 높이 44px, 최소 너비 100px
- **여백**: 좌우 16px (작은 화면: 12px)
