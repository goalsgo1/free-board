# 📐 화면 레이아웃 디자인 및 화면 이동 플로우

## 📱 반응형 레이아웃 가이드

모든 화면은 다양한 모바일 기기 크기에서도 레이아웃이 깨지지 않도록 반응형으로 설계되어야 합니다.

### 반응형 레이아웃 원칙
- **최소 화면 크기**: 320px × 568px (iPhone SE)
- **텍스트 오버플로우 방지**: 모든 텍스트는 `maxLines`와 `TextOverflow.ellipsis` 사용
- **제목 잘림 방지**: 제목은 최대 2줄까지, 필요시 "..." 표시
- **입력 필드**: 최소 높이 44px, 폰트 크기 최소 16px
- **버튼**: 작은 화면에서는 전체 너비 사용
- **이미지**: 화면 너비를 초과하지 않도록, 비율 유지
- **스크롤**: 모든 화면에서 `SingleChildScrollView` 사용

**자세한 반응형 레이아웃 가이드**: [RESPONSIVE_LAYOUT.md](./RESPONSIVE_LAYOUT.md) 참고

### 진입 경로
- **로그인 후 자동 이동**: 회원가입/로그인 완료 시
- **하단 네비게이션**: 홈 아이콘 클릭

### 사용 가능한 기능
1. **기념일 알림 카드 클릭** → 추모관 상세 화면 이동
2. **추천 추모글 카드 클릭** → 추모관 상세 화면 이동
3. **최근 등록된 추모관 카드 클릭** → 추모관 상세 화면 이동
4. **하단 네비게이션**:
   - 📚 추모관 목록 화면 이동
   - 🕊️ 기도 요청 화면 이동
   - 💬 감정 공유 게시판 이동
   - 👤 마이페이지 이동

### 반응형 레이아웃 규칙

#### 작은 화면 (320px ~ 375px)
- **AppBar 제목**: 16px, 최대 1줄, `TextOverflow.ellipsis`
- **앱의 목적과 마인드 섹션**:
  - 제목: 16px, 최대 1줄
  - 메인 메시지: 14px, 최대 2줄
  - 서브 메시지: 12px, 최대 2줄
  - 패딩: 12px
  - `Flexible` 위젯 사용
- **기념일 알림 카드**:
  - 높이: 100px (고정)
  - 패딩: 12px
  - 제목: 14px, 최대 1줄
  - 날짜: 12px
  - 버튼: 전체 너비
- **추모글 카드**:
  - 높이: 180px (고정)
  - 이미지: 100px × 100px (고정)
  - 제목: 14px, 최대 1줄, `TextOverflow.ellipsis`
  - 내용: 12px, 최대 2줄, `TextOverflow.ellipsis`
  - `Flexible` 위젯 사용
- **최근 등록된 추모관**:
  - 카드 너비: 120px (고정)
  - 이미지: 80px × 80px (고정)
  - 이름: 12px, 최대 1줄, `TextOverflow.ellipsis`
- **앱 기능 안내**:
  - 아이템 높이: 60px
  - 아이콘: 24px
  - 텍스트: 12px, 최대 1줄

#### 중간 화면 (375px ~ 414px)
- **AppBar 제목**: 18px
- **앱의 목적과 마인드 섹션**: 패딩 16px
- **기념일 알림 카드**: 높이 120px, 패딩 16px
- **추모글 카드**: 높이 200px, 이미지 120px × 120px
- **최근 등록된 추모관**: 카드 너비 140px

#### 큰 화면 (414px+)
- **추모글 카드**: 높이 220px, 이미지 140px × 140px

#### 공통 규칙
- **전체 레이아웃**: `SingleChildScrollView` 사용
- **카드**: `mainAxisSize: MainAxisSize.min` 사용
- **텍스트**: `Flexible` 또는 `Expanded` 사용
- **여백**: 좌우 16px (작은 화면: 12px)

### 사용 가능한 기능
1. **Step 1: 기본 정보**

   - 사진 추가 버튼 → 이미지 선택
   - 사진 미리보기
   - 사진 삭제 버튼

3. **Step 3: 동영상**
   - YouTube 링크 입력 또는
   - 동영상 업로드 버튼
   - 다음 버튼 → Step 4 이동

4. **Step 4: 편지 작성**
   - 텍스트 편지 입력 또는
   - 음성 편지 녹음
   - 다음 버튼 → Step 5 이동

5. **Step 5: 설정**
   - 생일 선택
   - 기일 선택
   - 공개/비공개 토글
   - 저장 버튼 → 추모관 저장 → 추모관 상세 화면 이동

#### 작은 화면 (320px ~ 375px)
- **AppBar 제목**: 16px, 최대 1줄, `TextOverflow.ellipsis`
- **Step Indicator**: 아이콘만 표시 (24px)
- **입력 필드**: 
  - 전체 너비 - 32px
  - 높이: 44px
  - 폰트: 16px
  - 패딩: 좌우 12px, 상하 12px
- **다중 줄 입력 필드**:
  - 최소 높이: 100px
  - 최대 높이: 화면 높이의 30%
  - 스크롤 가능
- **이미지 업로드**:
  - 그리드: 2열
  - 이미지 크기: (화면 너비 - 48px) / 2
  - 삭제 버튼: 24px
- **버튼**: 
  - 전체 너비
  - 높이: 48px
  - 폰트: 14px
  - 세로 배치
- **저장 버튼**: 하단 고정

#### 중간 화면 (375px ~ 414px)
- **Step Indicator**: 아이콘 + 텍스트
- **이미지 그리드**: 3열
- **버튼**: 가로 배치 가능

#### 공통 규칙
- **전체 레이아웃**: `SingleChildScrollView` 사용
- **입력 필드**: `TextEditingController` 사용
- **이미지**: `BoxFit.cover` 사용
- **여백**: 좌우 16px (작은 화면: 12px)

### 반응형 레이아웃 규칙

#### 작은 화면 (320px ~ 375px)
- **AppBar 제목**: 16px, 최대 1줄, `TextOverflow.ellipsis`
- **책장 컨테이너**: 전체 너비 - 32px, 패딩 12px
- **선반**: 높이 8px, 패딩 좌우 12px
- **책 버튼**:
  - 너비: (화면 너비 - 64px) / 책 개수 (최소 60px)
  - 높이: 120px
  - 폰트: 10px, 최대 1줄, `TextOverflow.ellipsis`
- **시간축**: 높이 40px, 폰트 12px
- **과거/미래 표시**: 12px, 최대 1줄

#### 중간 화면 (375px ~ 414px)
- **책 버튼**: 높이 140px, 폰트 12px
- **선반**: 패딩 좌우 16px

#### 큰 화면 (414px+)
- **책 버튼**: 높이 160px, 폰트 14px

#### 공통 규칙
- **전체 레이아웃**: `SingleChildScrollView` 사용
- **책장**: 가로 스크롤 가능
